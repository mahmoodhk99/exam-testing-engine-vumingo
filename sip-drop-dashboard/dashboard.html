📶 Nailer Drop Abuse Dashboard — sip:100@zain.com

Flags agents who drop the nailer from the softphone: a sip:100 drop with no matching In Job Break = abuse

What this dashboard finds

Agents who hang up the sip:100@zain.com nailer call from their softphone to dodge taking calls. When a nailer drop is legitimate, POM logs an In Job Break. A nailer drop with no matching In Job Break = abuse.

Words to know

  • Nailer call (sip:100@zain.com) — the always-on call that keeps an agent connected to the dialer; customer calls bridge into it.
  • In Job Break — POM’s record of a legitimate nailer drop while working a job.
  • Session — one agent login (Login → Logout).
  • Job — a campaign the agent attached to (Job Attach → Job Detach).

Top cards

  • Abusive drops (no break) — total nailer drops with no matching break.
  • Agents flagged / Sessions with abuse — how widespread it is.
  • Dropped w/o break → then off — sessions where an abuse drop was followed by going off before the next login.
  • sip:100 nailer drops / In Job Break drops — raw totals for reference.

The agent table

  • Abuse drops (no break) = nailer drops − in-job breaks (per agent).
  • Abuse sessions = how many of the agent’s sessions had any abuse.
  • Abuse time = total off-work time caused by abuse drops (from each drop to the next job they attached to; overlapping/parallel drops counted once).
  • sip:100 drops / In Job Breaks / Jobs / Talk (min) — supporting numbers.
  • Use the search, the sort menu (or click a column header), and Only flagged agents to focus. Click any agent row to expand the detail.

Inside an expanded agent

Each session has a header bar:
  • Nailer dropped: N times — nailer drops in that session.
  • In Job breaks: N — breaks POM logged that session.
  • Abuse time — off-work time from that session’s abuse drops.
  • Off after — time logged out after this session (next login − this logout).
  • ⚑ abuse: N drops w/o break the session had abuse drops.
  • ⚠ dropped w/o break, then off N before next login an abuse drop and then off before the next login — i.e. dropped the nailer to take an unlogged gap.
Below the header:
  • 🔴 Nailer dropped: <date> one row per drop — the drop’s date/time and how long the agent was off until the date of the next job they attached to.
  • The jobs list (by date/time): each campaign with Job Attach, Job Detach, Calls, Talk (min) and Breaks.

Colours

  • Red = abuse (drop with no break).
  • Amber = dropped w/o break, then went off before the next login.

How the key numbers are worked out

  • A nailer drop = a sip:100@zain.com call that ended in Far End Disconnect, placed into the session by its time.
  • abuse (per session) = nailer drops − in-job breaks.
  • abuse time = the merged spans from each abuse drop to the next job attach, so parallel drops are counted once (never more than the agent’s day).

How to use it day to day

Sort by Abuse drops or Abuse time, turn on Only flagged agents, open the worst agents, and read their red drop rows and amber sessions to see exactly when they dropped the nailer and how long they stayed off work.
